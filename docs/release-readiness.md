# iRead 发版就绪度

## 当前结论

当前版本定位为 `0.2.0-beta.2`。它已适合发布到 GitHub 供本地测试者安装和反馈，但还不应标记为稳定版。

## 已具备

- 任意单领域或多领域订阅模型，不包含领域白名单。
- Codex 批量信源检索、提案生成、严格校验和人工批准边界。
- WorkBuddy 原生提案流程，不依赖另一台机器安装 Codex。
- 多领域主题合并、共用信源去重、独立运行目录和统一报告策略。
- 日报、周报、月报生成，以及 RSS、微信公众号、Notion 和公开归档能力。
- MIT 许可证、第三方组件与信源披露、全文导出权利确认开关。
- 用户批准后的本地微信二维码授权、信源匹配、最近一个自然月回补、就绪验收和定时任务。
- 无公众号后台权限时的 RSS/网页源降级模式，以及不含第三方全文的定期本地公开归档。
- 必抓 `web_pending` 信源未连接时使用 `active_with_gaps` 并产生完整性告警，不把候选源误报为已采集。

## 稳定版前仍需完成

- 在一台干净电脑上完成 WorkBuddy 安装和真实多领域验收。
- 建立首个 Git 提交、GitHub 远端、版本标签和发布说明。
- 对公开仓库做最终密钥扫描，并确认运行数据与第三方正文没有被提交。
- 如需让 GitHub 自动公开本地归档，还需由仓库所有者明确开启并配置有限的推送凭据；beta 默认只定时生成本地元数据归档。
- 明确源码安装是首发分发方式；当前不承诺独立 PyPI wheel 能脱离仓库资源运行。

## 发版门槛

本机 Codex 验收通过后，可以推送 GitHub 并发布 `v0.2.0-beta.2`。另一台电脑的 WorkBuddy 安装、微信扫码或 RSS 降级路径、回补与报告验收通过后，再评估 `v0.2.0` 稳定版。
