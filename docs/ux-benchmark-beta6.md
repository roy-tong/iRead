# Beta 6 第三方用户体验测试

测试日期：2026-07-20。测试把宿主 Agent、GitHub 网络、iRead 安装器和研究管线分开计时，避免把网络故障误判为 Python 安装慢。

| 信源质量 | 只要总数和基本角色格式可被应用 | 应用前强制角色配比、可采集率、证据 URL 和代表作去重 |

## 跨 Agent 实测边界

- **Codex**：插件 `0.2.0-beta.6` 已安装并被 `plugin list` 识别。Beta 5 的默认提示超数警告已消失。非交互 `codex exec` 在测试时遭遇模型流断开，因此未将该次失败计入 iRead 命令或结果评分。
- **Claude Code**：未安装时 `/iread` 正确返回未知命令；安装后调试日志确认用户 Skill 被加载。当前本机 Claude Code 未登录，未完成模型端的订阅研究。
- **豆包专业版**：已验证通用 Agent Skill 目录和 ZIP 产物，但没有可自动化的公开 CLI 用于验证宿主内实际召起，仍为实验性适配。

## 信源推荐验收

用已完成的“医疗器械监管 + 新能源电力市场 + 独立游戏发行”第三方测试方案重新验收：

- 27 个候选源，平均每领域 9 个；15 个 RSS 可自动采集，12 个为 `web_pending`。
- 独立游戏通过新门槛：官方、独立报道、专业分析、从业者和发现线索都有覆盖，67% 可自动采集。
- 医疗器械被拦截：缺专家/从业者、发现型线索，且只有 1 个独立核验源。
- 新能源被拦截：缺专家/从业者和发现型线索。

这说明旧版“官方机构 + 垂直媒体”的名单表面上权威，但不足以支撑实践经验、弱信号和独立核验。Beta 6 不再允许这类偏科方案直接进入采集。

## 采集与日报

真实 RSS 测试使用最近一个自然月边界。15 源并发请求在 10.655 秒返回，导入 318 篇，跳过边界前 662 条；EIA 单源超时，Beta 6 已增加一次有界重试。

已生成的 AI/具身日报共 5096 个字符，包含 3 条当日判断、5 篇精读、5 条扫读、3 个原创观点、3 个待验证争议和 5 个下一步动作。Beta 6 质量评分为 94/100，唯一扣分项是结构化分析覆盖率只有约 60%。优点是区分事实、公司口径和推断，并为用户做出精读/跳过决策。限制是该期包含人工编辑校订，不能作为“全自动日报已稳定超过预期”的证据。

Beta 6 通过并行分析、8000 字分布式文章抽样和中等强度富化推理降低日报前的时间与 Token 压力；报告综合和信源研究仍保持高推理强度。

## 当前发版判断

Beta 6 已适合继续扩大 Codex 和 Claude Code 公开测试。在以下三项完成前，不应发布稳定版：

1. 在一台已登录的干净 Claude Code 电脑和真实豆包客户端完成从安装到报告的端到端路径。
2. 至少三个非 AI 领域在无人工补写情况下连续一周按时产出报告。
3. 将 Beta 6 已有的篇幅、引用、分析覆盖和栏目上限评分，扩展到主题偏科、事件重复率和用户长期反馈。
